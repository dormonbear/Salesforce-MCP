    **Step 2: Implement disk-persistence.ts**

    Create packages/mcp-provider-dx-core/src/schema/disk-persistence.ts:

    ```typescript
    import { mkdir, readFile, writeFile, readdir } from 'node:fs/promises';
    import { join } from 'node:path';
    import { SchemaEntry } from './types.js';
    ```

    Class `SchemaDiskPersistence`:

    - **Constructor**: `{ dataDir: string, ttlMs: number, debounceMs?: number }`.
      - `cacheDir = join(dataDir, 'schema-cache')` (per D-15)
      - `debounceMs` defaults to `5000` (per D-14)
      - `pendingTimer: ReturnType<typeof setTimeout> | null = null`
      - `pendingOrgs: Set<string>` — tracks which orgs have dirty data

    - **`async saveOrg(orgUsername: string, entries: Map<string, SchemaEntry>): Promise<void>`**:
      1. `await mkdir(this.cacheDir, { recursive: true })`
      2. Build serializable object: `{ entries: Object.fromEntries(entries), savedAt: Date.now() }`
      3. `const filePath = join(this.cacheDir, orgUsername + '.json')`
      4. `await writeFile(filePath, JSON.stringify(data, null, 2), 'utf-8')`

    - **`async loadOrg(orgUsername: string): Promise<Map<string, SchemaEntry>>`**:
      1. `const filePath = join(this.cacheDir, orgUsername + '.json')`
      2. Try to read file. If ENOENT, return `new Map()`.
      3. Parse JSON. If parse fails, return `new Map()`.
      4. Filter entries: keep only where `(now - entry.cachedAt) <= this.ttlMs`.
      5. Return `new Map(Object.entries(filtered))`.

    - **`async loadAll(): Promise<Map<string, Map<string, SchemaEntry>>>`**:
      1. Try `readdir(this.cacheDir)`. If ENOENT, return `new Map()`.
      2. Filter files ending in `.json`.
      3. For each file, extract orgUsername (strip `.json`), call `loadOrg(orgUsername)`.
      4. Return Map of all org entries (skip empty maps).

    - **`scheduleSave(orgUsername: string, getEntries: () => Map<string, SchemaEntry>): void`**:
      1. `this.pendingOrgs.add(orgUsername)`
      2. Store `getEntries` callback reference.
      3. If timer already set, do nothing (debounce coalescing).
      4. Set timer: `this.pendingTimer = setTimeout(() => this.executeSave(), this.debounceMs)`

    - **`async flush(): Promise<void>`**:
      1. Clear pending timer if set.
      2. Execute save for all pending orgs immediately.
      3. Clear pendingOrgs set.

    - **Private `async executeSave(): Promise<void>`**:
      1. Copy pendingOrgs to local set, clear it.
      2. For each org in set, call `saveOrg(org, getEntries())`.
      3. Wrap in try/catch — disk write failures are non-fatal (log warning, do not throw).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk → in-memory cache | JSON files loaded from dataDir could be tampered with if dataDir is writable by other processes |
| orgUsername → file path | orgUsername used in file path construction; must not allow path traversal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05 | Tampering | disk-persistence.ts loadOrg() | accept | dataDir is server-controlled (same trust as server config); corrupted files gracefully return empty Map |
| T-10-06 | Information Disclosure | JSON cache files | accept | Cache files contain schema metadata (field names, not record data); stored in server dataDir alongside existing MCP data; same trust level |
| T-10-07 | Elevation of Privilege | File path from orgUsername | mitigate | orgUsername from Connection.getUsername() is a Salesforce email format (user@domain); validate no path separators (/ or \\) exist in username before constructing file path; reject with empty Map if validation fails |
| T-10-08 | Denial of Service | Debounce timer | accept | Debounce limits disk writes to at most once per 5 seconds; worst case is one write per 5s per org which is negligible I/O |
</threat_model>
