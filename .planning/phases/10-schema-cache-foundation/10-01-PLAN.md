    Define the following types:

    ```typescript
    export const SchemaEntryType = {
      FullDescribe: 'full-describe',
      PartialFields: 'partial-fields',
      RelationshipEdges: 'relationship-edges',
    } as const;

    export type RelationshipEdge = {
      from: string;
      to: string;
      via: string;
      type: 'lookup' | 'master-detail';
    };

    export type FullDescribeEntry = {
      type: typeof SchemaEntryType.FullDescribe;

      cachedAt: number;
    };

    export type PartialFieldsEntry = {
      type: typeof SchemaEntryType.PartialFields;
      objectName: string;
      fieldNames: string[];
      cachedAt: number;
    };

    export type RelationshipEdgesEntry = {
      type: typeof SchemaEntryType.RelationshipEdges;
      edges: RelationshipEdge[];
      cachedAt: number;
    };

    export type SchemaEntry = FullDescribeEntry | PartialFieldsEntry | RelationshipEdgesEntry;
    ```

    Use `Record<string, unknown>` for DescribeSObjectResult data to avoid importing @salesforce/core types into the type file. The actual typing will be enforced at the call site in later phases.

    **Step 3: Create schema-service.ts at packages/mcp-provider-dx-core/src/schema/schema-service.ts**

    Implement SchemaService class:

    - **Constructor**: accepts `{ ttlMs?: number }` options. Read `SF_SCHEMA_CACHE_TTL_MINUTES` env var. If set, use `parseInt(value) * 60_000`. Otherwise use the `ttlMs` option, defaulting to `3_600_000` (1 hour per D-07).
    - **Private state**:
      - `orgCaches: Map<string, LRUCache<string, SchemaEntry>>` (per D-06)
      - `inFlight: Map<string, Promise<SchemaEntry>>` (per D-09, D-10)
      - `ttlMs: number` (resolved TTL)
    - **Private helper `getOrCreateOrgCache(orgUsername: string)`**: returns the LRU cache for an org, creating one if needed. LRU config: `{ max: 100, ttl: this.ttlMs }` (per D-08).
    - **`get(orgUsername: string, objectName: string): SchemaEntry | undefined`**: normalizes objectName to lowercase, looks up in org's LRU. Returns undefined on miss.
    - **`set(orgUsername: string, objectName: string, entry: SchemaEntry): void`**: normalizes objectName to lowercase, sets in org's LRU.
    - **`describeAndCache(orgUsername: string, objectName: string, describeFn: () => Promise<SchemaEntry>): Promise<SchemaEntry>`**:
      1. Check cache first: `const cached = this.get(orgUsername, objectName); if (cached) return cached;`
      2. Build flight key: `const flightKey = orgUsername + ':' + objectName.toLowerCase();`
      3. Check in-flight: `if (this.inFlight.has(flightKey)) return this.inFlight.get(flightKey)!;`
      4. Create promise: `const promise = describeFn().then(entry => { this.set(orgUsername, objectName, entry); return entry; }).finally(() => { this.inFlight.delete(flightKey); });`
      5. Store in-flight: `this.inFlight.set(flightKey, promise);`
      6. Return promise.
    - **`invalidate(orgUsername: string, objectName: string): boolean`**: removes entry from org's LRU. Returns true if deleted.
    - **`invalidateOrg(orgUsername: string): void`**: deletes entire org cache.
    - **`clear(): void`**: clears all org caches.
    - **`getOrgCacheSize(orgUsername: string): number`**: returns number of entries in org's LRU (for testing).
    - **`getAllOrgUsernames(): string[]`**: returns list of orgs with cache entries (needed for disk persistence in Plan 02).

    Expose a callback hook for persistence: `onMutation?: () => void` — called after every set/invalidate. Plan 02 will use this for debounced disk writes.

    Object name normalization: always `.toLowerCase()` for cache keys (per Pitfall 11: case-insensitive SOQL but case-sensitive keys).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| org identity → cache partition | Cache key derived from orgUsername must prevent cross-org data access |
| describeFn callback → cache storage | External describe results stored as-is; no untrusted user input enters cache keys |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Information Disclosure | SchemaService.get() | mitigate | Cache keyed by canonical orgUsername (D-05); test verifies get("orgA", obj) never returns orgB data |
| T-10-02 | Denial of Service | LRUCache per-org | mitigate | LRU max=100 per org (D-08); prevents unbounded memory growth from describe storms |
| T-10-03 | Spoofing | Cache key from orgUsername | accept | orgUsername comes from Connection.getUsername() which is already authenticated; no user-supplied aliases used as cache keys |
| T-10-04 | Tampering | SchemaEntry data | accept | Cache is in-process memory; no external write surface; entries come only from describeFn callback |
</threat_model>
