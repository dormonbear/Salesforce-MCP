<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tool catch block -> LLM context | Error text from SfError.message is returned to LLM; must not leak stack traces or internal paths |
| SfError.actions -> recovery text | Actions array from @salesforce/core is surfaced verbatim; these are user-facing by design |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03 | Information Disclosure | SfError.message in toolError | accept | SfError.message is user-facing by design in @salesforce/core. No internal paths or secrets. Stack traces never included (toolError only takes message string). |
| T-07-04 | Denial of Service | Recovery hints suggesting retry | mitigate | System errors (network, auth) recovery hints say "inform the user" not "retry", preventing infinite loops. classifyError() routes these correctly. |
| T-07-05 | Spoofing | Fabricated recovery hints | accept | Recovery hints are hardcoded strings or from SfError.actions (trusted source). No user input flows into recovery text. |
</threat_model>
