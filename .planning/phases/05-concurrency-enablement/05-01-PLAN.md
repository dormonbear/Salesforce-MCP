# Phase 05 Plan 01: Remove Global Mutex and Enable Parallel Tool Execution

**Goal**: Remove the global toolExecutionMutex and enable concurrent tool execution, with targeted serialization for lwc-experts tools.

## Changes

### 1. auth.ts — Remove process.cwd() dependency (CONC-01)
- Removed `ConfigAggregator.clearInstance(process.cwd())` from `getDefaultConfig()`
- CWD is now constant since all process.chdir() calls removed — singleton stays valid
- ConfigAggregator.create() returns cached singleton without clearing/recreating

### 2. sf-mcp-server.ts �� Remove global Mutex (CONC-02), add targeted lock (CONC-03)
- Replaced `toolExecutionMutex` (global) with `serializedToolMutex` + `serializedTools` set
- Added `markToolAsSerialized(name)` method
- Middleware dispatches: serialized tools go through Mutex, all others run concurrently as independent Promises

### 3. registry-utils.ts — Mark lwc-experts tools (CONC-03)
- After registering each tool, checks if it belongs to `Toolset.LWC_EXPERTS`
- If so, calls `server.markToolAsSerialized(toolName)` — these tools have internal process.chdir()

### 4. Tests — Concurrent stress test (CONC-04, CONC-05)
- `should run normal tools concurrently (interleaved)` — verifies no serialization for standard tools
- `should serialize tools marked as serialized` — verifies lwc-experts-style tools still serialize
- `should preserve correct args for each concurrent call` — verifies no arg cross-contamination
- `should handle 5+ concurrent tool calls without errors` — 7 tools in parallel, all complete
