```typescript
// Graceful shutdown on SIGTERM (replaces broken process.stdin.on('SIGTERM'))
// per MCP lifecycle spec: https://modelcontextprotocol.io/specification/2025-06-18/basic/lifecycle#stdio
process.on('SIGTERM', () => {
  this.telemetry?.sendEvent('SERVER_STOPPED_SUCCESS');
  this.telemetry?.stop();
  // Force exit after 5 seconds if graceful close hangs
  const forceExit = setTimeout(() => {
    process.exit(1);
  }, 5000);
  forceExit.unref(); // Don't block event loop if everything closes cleanly
  server.close().then(() => {
    process.exit(0);
  }).catch(() => {
    process.exit(1);
  });
});
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS signal → process | SIGTERM is a trusted OS signal but incorrect handler means it is silently ignored |
| tool name → category lookup | Tool names come from the MCP protocol; unknown names default to 'write' which is safe-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Denial of Service | SIGTERM handler | mitigate | Replace process.stdin.on('SIGTERM') with process.on('SIGTERM') so the signal is actually handled; 5-second forced-exit timeout prevents indefinite hang |
| T-02-05 | Elevation of Privilege | tool-categories missing entries | mitigate | Missing entries default to 'write' in getToolCategory() — any read-only org would be incorrectly blocked from read operations; adding all entries with correct categories fixes the over-restriction |
| T-02-06 | Information Disclosure | graceful shutdown telemetry flush | accept | Telemetry events include tool names but no PII; flushing on SIGTERM is intentional and consistent with existing stdin close behavior |
</threat_model>
