    Implementation strategy (per D-01, regex-based):

    1. **Guard:** If `query` is falsy or only whitespace, return null.

    2. **Bail-out checks for complex queries (per D-03):**
       - Subqueries: check for `(SELECT` pattern (open paren immediately or with spaces before SELECT) using case-insensitive match. Use `/\(\s*SELECT\b/i` — this avoids false positives from string literals like `WHERE Name = 'SELECT FROM'` because `(SELECT` specifically indicates a subquery.
       - Aggregates: check for aggregate function patterns `/\b(COUNT|SUM|AVG|MIN|MAX)\s*\(/i`
       - GROUP BY: check for `/\bGROUP\s+BY\b/i`
       - TYPEOF: check for `/\bTYPEOF\b/i`
       - HAVING: check for `/\bHAVING\b/i`
       If any match, return null.

    3. **Extract SELECT...FROM via regex (per D-01):**
       Use case-insensitive regex: `/^\s*SELECT\s+([\s\S]+?)\s+FROM\s+(\w+)/i`
       - Group 1 captures the field list between SELECT and FROM
       - Group 2 captures the object name (first word after FROM — `\w+` matches `Account`, `Custom_Object__c` etc.)
       If no match, return null.

    4. **Parse field list (per D-05):**
       - Split group 1 on commas
       - For each field token: trim whitespace, then take only the first word (split on whitespace, take index 0) — this strips aliases (`Name n` → `Name`, `Industry i` → `Industry`)
       - Filter out dotted names (containing `.`) — these are relationship traversals per D-03
       - Filter out empty strings

    5. **Final check:** If fieldNames array is empty after filtering (all fields were dotted), return null.

    6. Return `{ objectName, fieldNames }`.

    **CRITICAL — ReDoS prevention:** All regex patterns must be simple and non-backtracking. The `[\s\S]+?` in the main regex uses a lazy quantifier bounded by the `FROM` keyword — this is safe. Do NOT use nested quantifiers like `(a+)+` or `(a*)*`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User SOQL → regex parser | Untrusted SOQL string from AI agent is parsed by regex; malformed input must not crash or hang |
| Parsed fields → schema cache | Field names extracted from user query are stored in per-org isolated LRU cache |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
